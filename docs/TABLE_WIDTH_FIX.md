# 🎨 报价模式表格美化优化

## 🎯 优化目标

让报价模式下的表格表头蓝色横条看起来更完整、更美观。

---

## 📊 问题分析

### 优化前
```
账单模式（8列）：
┌────────────────────────────────────────────────────────┐
│ 舱房 │ PAX │ 直客价 │ 净船票 │ 佣金 │ 税费 │ HSC │ 结算价 │
└────────────────────────────────────────────────────────┘
100% 宽度，蓝色横条完整

报价模式（隐藏3列后）：
┌─────────────────────────────────┐
│ 舱房 │ PAX │ 直客价 │ 税费 │ HSC │              空白
└─────────────────────────────────┘
约52%宽度，蓝色横条不完整 ❌
```

### 问题
- 隐藏列后，剩余列宽度不变
- 表格右侧留白过多
- 蓝色表头横条看起来短小、不美观

---

## ✅ 解决方案

### 调整策略
在报价模式下，重新分配列宽，让5个可见列均匀填满整个表格宽度。

### 列宽分配

| 列名 | 账单模式 | 报价模式 | 说明 |
|------|----------|----------|------|
| 舱房说明 | 34% | **45%** ↑ | 加宽11%，更多空间显示舱房信息 |
| 人数/数量 | 8% | **12%** ↑ | 加宽4%，数字更清晰 |
| 直客价 | 10% | **15%** ↑ | 加宽5%，价格更醒目 |
| 税费 | 9% | **14%** ↑ | 加宽5%，更易阅读 |
| 服务费 | 9% | **14%** ↑ | 加宽5%，更易阅读 |
| **总计** | **70%** | **100%** | 填满整个宽度 ✅ |

## 💻 技术实现

### CSS 代码

```css
/* 报价模式下调整表格列宽 */
.quote-mode .inv-table th:nth-child(1) { width: 45% !important; }  /* 舱房说明 */
.quote-mode .inv-table th:nth-child(2) { width: 12% !important; }  /* 人数 */
.quote-mode .inv-table th:nth-child(3) { width: 15% !important; }  /* 直客价 */
.quote-mode .inv-table th:nth-child(6) { width: 14% !important; }  /* 税费 */
.quote-mode .inv-table th:nth-child(7) { width: 14% !important; }  /* 服务费 */

/* 对应的td也需要调整 */
.quote-mode .inv-table td:nth-child(1) { width: 45% !important; }
.quote-mode .inv-table td:nth-child(2) { width: 12% !important; }
.quote-mode .inv-table td:nth-child(3) { width: 15% !important; }
.quote-mode .inv-table td:nth-child(6) { width: 14% !important; }
.quote-mode .inv-table td:nth-child(7) { width: 14% !important; }
```

### 工作原理

1. **CSS选择器** - 使用 `:nth-child()` 精确选择特定列
2. **报价模式特定** - 仅在 `.quote-mode` 下生效
3. **强制覆盖** - 使用 `!important` 确保优先级
4. **表头和内容统一** - th 和 td 使用相同宽度

## 📝 技术细节

### nth-child 选择器说明

```css
/* 第1列 - 舱房说明（始终显示）*/
:nth-child(1)

/* 第2列 - 人数（始终显示）*/
:nth-child(2)

/* 第3列 - 直客价（始终显示）*/
:nth-child(3)

/* 第4列 - 净船票（隐藏）*/
/* 第5列 - 佣金（隐藏）*/

/* 第6列 - 税费（始终显示）*/
:nth-child(6)

/* 第7列 - 服务费（始终显示）*/
:nth-child(7)

/* 第8列 - 结算价（隐藏）*/
```

### 为什么不是连续的？

因为隐藏的列（第4、5、8列）虽然不显示，但仍然存在于DOM中，占用 nth-child 的位置。
